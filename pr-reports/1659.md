# PR 1659 Report: Add 6529 Help Bot V1 Runtime

## PR

- GitHub PR: https://github.com/6529-Collections/6529seize-backend/pull/1659
- Branch: `codex/6529-help-bot-spec`
- Target: `main`
- Related frontend PR: https://github.com/6529-Collections/6529seize-frontend/pull/2786

## Task Summary

Implement V1 of `@6529help` without full RAG: fast trigger detection, visible bot status reactions, reply drops, cached frontend help-index retrieval with canonical URLs, optional Bedrock wording, and reliable failure replies.

## Backend Change Summary

- Added a post-create drop hook that detects explicit `@6529help` mentions and direct replies to prior bot replies.
- Added `help_bot_interactions` for idempotency, status, auditability, and retry-safe worker processing.
- Added `helpBotReplyLoop`, backed by SQS `help-bot-replies` plus DLQ, to answer asynchronously.
- Added cached frontend help-index retrieval from `/help-index.json`, with `HELP_BOT_INDEX_URL` and `HELP_BOT_INDEX_CACHE_TTL_MS` overrides.
- Kept V1 retrieval deterministic with alias/keyword matching over the frontend-owned corpus.
- Added optional Bedrock rendering behind an internal renderer; deterministic record answers remain the fallback when Bedrock is unset or fails.
- Added internal bot reaction/drop writers for 👀, ✅, ⚠️, bot replies, no-reliable-source replies, and technical-failure replies.
- Updated deployment config generation and architecture/runtime docs for the new API-to-SQS-to-worker flow.

## Backend Impact Summary

- New deployable service: `helpBotReplyLoop`.
- Existing deployable service changed: `api`.
- DB/entity changes: new TypeORM entity/table `help_bot_interactions`.
- Queue changes: new SQS queue `help-bot-replies` and `help-bot-replies-dlq`.
- External integration: optional Amazon Bedrock invocation when `HELP_BOT_BEDROCK_MODEL_ID` is configured.
- Public API contract changes: none.

## Env/Config Changes

- `HELP_BOT_ENABLED`: enables runtime behavior when true.
- `HELP_BOT_PROFILE_ID`: profile id of the manually created `@6529help` bot identity.
- `HELP_BOT_SQS_URL`: API queue URL; deploy workflow now derives it as `help-bot-replies`.
- `HELP_BOT_BEDROCK_MODEL_ID`: optional Bedrock model id for natural wording.
- `HELP_BOT_BASE_URL`: optional canonical URL base, default `https://6529.io`.

Deploying this code alone leaves the bot inactive unless the enable/profile env vars are set.
