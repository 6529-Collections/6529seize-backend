# PR 1659 Report: Add Help6529 Bot V1 Runtime

## PR

- GitHub PR: https://github.com/6529-Collections/6529seize-backend/pull/1659
- Branch: `codex/6529-help-bot-spec`
- Target: `main`
- Related frontend PR: https://github.com/6529-Collections/6529seize-frontend/pull/2786

## Task Summary

Implement V1 of `@help6529` for public Waves: mention detection, visible bot reactions, threaded replies, follow-up handling, frontend-owned knowledge retrieval, bounded public-data lookups, REP credit gating, and safe fallback behavior without full RAG.

## Backend Change Summary

- Added a post-create drop hook that detects explicit `@help6529` mentions, mention-only replies to another user's question, and replies to prior bot messages.
- Added `help_bot_interactions` for idempotent queueing, target-drop tracking, processing state, and retry-safe worker execution.
- Added `help_bot_credit_events` for `Help6529 Credits` REP grants, spends, and refunds.
- Added `helpBotReplyLoop`, backed by SQS `help-bot-replies` plus DLQ, to generate answers asynchronously and post threaded replies.
- Added frontend-owned help-index loading from `/help-index.json`, with environment-matched staging/prod frontend URLs and deterministic retrieval over aliases, keywords, facts, and labels.
- Added Bedrock/Claude answer rendering with deterministic fallback; Bedrock does not produce executable SQL.
- Added a backend-owned public-data catalog and compiler for bounded aggregate lookups.
- Added profile TDH public-data support so questions like "who has the highest TDH currently" answer with the top profile and value instead of falling back to Meme Card TDH-rate content.
- Hardened follow-up intent handling so corrections like "I meant which profile" can switch from card metrics to profile TDH.
- Fixed public-data links so single Meme Card metric answers link the actual `/the-memes/<id>` route and top-profile answers link the real profile route.
- Fixed top-profile TDH link labels so the visible Markdown link text is the actual returned `@handle`, not the generic "Highest Profile TDH" query title.
- Added a frontend-calendar answer path for Meme Card drop timing questions. The worker detects next/current/specific Meme Card timing questions and calls the environment-matching frontend calendar API before falling back to static help-index retrieval.
- Treats bare `card`/`cards` phrasing as Meme Card context, so questions like "when is card 1234 dropping" resolve through the calendar answer path instead of being rejected as out of scope.
- Suppressed placeholder corpus links such as `/{user}` so bot replies no longer emit broken profile URLs.
- Resolved tech-team handles through identity profiles and sends real `mentioned_users` metadata; no-source replies now use natural referral copy before tagging the team.
- Excluded restricted-visibility waves and DMs before queueing or calling Bedrock.
- Added per-user spam suppression before queueing.
- Added deterministic boundaries for impossible grants, prompt-injection attempts, and private-data pokes.
- Audited the bot-handle rename; only `help6529` remains in the backend branch, with no deprecated-handle literals retained.

## Backend Impact Summary

- New deployable service: `helpBotReplyLoop`.
- Existing deployable service changed by the full PR: `api`.
- DB/entity changes in the full PR: `help_bot_interactions`, `help_bot_credit_events`.
- Queue changes: `help-bot-replies`, `help-bot-replies-dlq`.
- Public API contract changes: none.
- External read integration: `helpBotReplyLoop` now reads the environment-matching frontend Memes calendar API for drop timing answers.
- Sensitive-data boundary: restricted/private/DM content is skipped before the bot answer path.

## Env/Config Changes

- Required: none. The bot activates when the `help6529` profile exists and can be resolved.
- Optional: `HELP_BOT_TECH_TEAM_HANDLES`, a comma-separated or semicolon-separated list of handles to mention for unanswered 6529 product questions.
