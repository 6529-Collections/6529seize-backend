## Task Summary
Fix inconsistent hide/restore link-preview behavior in wave posts by making the backend endpoint accept an explicit final `hide_link_preview` state, while keeping legacy no-body toggle behavior.

## Backend Change Summary
- Added optional request body support for `POST /drops/{dropId}/toggle-hide-link-preview`.
- Added generated `ApiToggleHideLinkPreviewRequest` model.
- Preserved no-body legacy toggle semantics.
- Uses a conditional DB update and gates websocket notification on actual affected rows, so explicit hide/restore requests remain deterministic even if another request changes the row after the authorization read.
- Added focused service tests for legacy toggle, explicit hide, explicit restore, and explicit no-op.
