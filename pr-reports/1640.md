## Summary

Adds category-wide Profile REP and Wave REP analytics endpoints, then hardens the Wave REP visibility path so users and proxies only see REP for waves they can read.

## What Changed

- Added global REP category endpoints for suggested categories, category overview, ratings, recipients, givers, Wave REP overview, waves, and wave contributors.
- Scoped Wave REP visibility through the shared wave read-context helper so proxy sessions without `READ_WAVE` do not inherit the role profile's private wave groups.
- Added an explicit direct-message guard to the wave and parent-wave SQL visibility filter so malformed or legacy DMs cannot be treated as public when `visibility_group_id` is null.
- Added focused unit coverage for normal viewers, proxy sessions without `READ_WAVE`, proxy sessions with `READ_WAVE`, and generated SQL visibility predicates.
