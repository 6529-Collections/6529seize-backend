## Task Summary

Add backend analytics so the frontend can show category-wide Profile REP and Wave REP details from one selected REP category, including waves that use the category and contributors inside those waves.

## Backend Changes

- Adds category-wide Wave REP analytics endpoints so the frontend can show waves using a REP category without requiring a second wave search.
- Adds suggested REP categories for the category search landing state.
- Sorts profile and wave REP rankings by absolute REP impact, with signed value used as a tie-breaker.
- Keeps Wave REP visibility scoped through existing wave and parent-wave visibility rules.
- Fixes follow-up Sonar findings by replacing nested ternary order/tie-breaker selection with explicit helper methods.
