# PR Report: GIF resize fallback

PR: https://github.com/6529-Collections/6529seize-backend/pull/1752
Branch: codex/s3-uploader-gif-sharp-fallback -> main
Related PRs: None

## Summary

Preserves the established ImageScript GIF resize path while adding an animated Sharp fallback for valid GIFs that ImageScript cannot decode.

## What Changed

- GIF resizing retries with Sharp's animated-image support only after ImageScript fails.
- A failure from both resizers now reaches the SQS partial-batch handler instead of acknowledging a job with missing derivatives.
- Focused coverage protects the primary path, fallback path, and terminal failure behavior.

## Backend Impact

- New services: None
- Changed services: `s3Uploader`
- Required service order or ownership: Redeploy `s3Uploader`; no ordering dependency.
- API: Not affected
- DB/entities/migrations: Not affected
- Queues/events/integrations: Existing SQS partial-batch retry and redrive behavior now applies when both GIF resizers fail.

## Config / Env

No changes.

## Release Notes

Animated GIF derivatives can be generated for assets that trigger ImageScript decoder failures, without changing the established processing path for GIFs that already work.
