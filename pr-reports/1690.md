# PR Report: TDH Recorded Flag For Meme Extended Data

PR: https://github.com/6529-Collections/6529seize-backend/pull/1690
Branch: `codex/recorded-in-tdh` -> `main`
Related PRs: https://github.com/6529-Collections/6529seize-frontend/pull/2983

## Summary

Adds an explicit `recorded_in_tdh` state to The Memes extended data so newly minted Memes that are not yet present in TDH data can remain unranked until TDH has recorded them.

## What Changed

- Added nullable `recorded_in_tdh` and `ranked_collection_size` fields to The Memes extended data entity.
- Added a TDH existence lookup against `tdh_nft` for each Meme when extended data is generated.
- Excluded Memes with `recorded_in_tdh === false` from extended-data rank assignment, leaving their rank fields at the existing unranked sentinel value.
- Stored the rank denominator as the number of Memes included in TDH rankings.
- Kept Meme Lab extended-data ranking unchanged and independent of TDH eligibility.
- Added focused coverage for recorded/unrecorded Meme ranking and Meme Lab ranking behavior.

## Backend Impact

- Changed services: NFT extended-data loop and `/api/memes_extended_data` response shape.
- API: The Memes extended data may now include `recorded_in_tdh` and `ranked_collection_size`.
- DB/entities/migrations: Adds nullable TypeORM entity fields on `memes_extended_data`; no migration file.
- Queues/events/integrations: None.

## Config / Env

None.

## Release Notes

New Memes that are not yet recorded in TDH will no longer participate in extended-data rank calculations until TDH data exists for the card.

## Risks / Follow-Ups

Existing rows return legacy behavior until the extended-data loop refreshes and writes the new fields.
